Getting Started

  1. Open Setup and select your Haldex generation.
  2. Calibrate the learned Haldex output table while Haldex CAN data is live.
  3. Save your settings. The Save button writes generation, mapped inputs, dashboard slots, and trigger settings.
  4. Return to Home and select a control mode. Basic setup is complete.

Requested Lock

OpenHaldex-S3 does not mechanically force the coupling by itself. It sends a requested Haldex lock value based on the active mode, mapped inputs, saved curves, and safety gates.

  • 0% means minimum requested lock.
  • 100% means maximum requested lock.
  • Values between saved points are interpolated smoothly.

Mode Settings

Map, Speed, Throttle, and RPM modes share the same behavior controls. The Enable toggles apply or bypass each gate while keeping the typed values saved.

Disengage under speed
Suppresses lock request below the selected vehicle speed. Full throttle bypasses this gate for launch control.
Minimum throttle
Requires pedal input to reach the selected percentage before the mode can request lock.
Disable above speed
Forces the lock request off above the selected vehicle speed.
Lock release ramp
Limits how quickly requested lock can fall, in percent per second. Lower values release more gently; higher values release faster.
Broadcast Haldex status
Publishes Haldex status back to the chassis bus for pages and vehicle modules that use that information.
Enable controller output
Allows OpenHaldex-S3 to send its control request. Turning this off leaves the controller in stock/pass-through behavior.
Home

Home is the driving screen. Use it to select the active AWD mode and watch the key status indicators.

  • Lock applies a fixed requested front/rear split from the ratio slider.
  • Throttle, Speed, RPM, and Map use the saved curve or map for that mode.
  • Off returns control to stock/pass-through behavior.
  • TMP shows Haldex temperature protection, CAN shows bus health, and AWD shows coupling activity/open state.
  • Dashboard tiles display default telemetry or the custom signals selected on Setup.
Setup

Setup is where the controller learns what vehicle generation and decoded CAN signals it should use.

  • Select the Haldex generation first, then Save. This applies the recommended defaults for Gen 1, 2, 4, or 5.
  • Use Calibrate to build the learned Haldex output table after Haldex CAN data is live.
  • Use the Decoded Signal Browser to assign a signal to an input, dashboard slot, or CAN mode trigger.
  • Mapped inputs feed the control modes. Advanced users can assign any decoded CAN signal to the Speed or Throttle input slots and use those as map axes.
  • The CAN Mode Trigger can enable one mode when one decoded signal matches the selected condition.
  • Clear All returns Setup selections to defaults, clears dashboard slots, resets the trigger, and clears learned calibration.
Map Mode

Map mode applies a requested lock value from a two-axis table. By default the axes are Speed and Throttle, but those input slots can be assigned to any decoded CAN signal in Setup.

  • Select a saved map from the dropdown, then click Load.
  • Click table cells to fine tune requested lock values.
  • Click a column header, then use Column Shaping to quickly rough in a custom map.
  • Edit Speed and Throttle bins to create a fully custom axis layout.
  • Use Save to write the active map to memory, or enter a name and use Save As to create a custom map.
  • Use Download TXT and upload TXT to share maps or move them between controllers.
  • Mode Behavior controls apply gates such as low-speed disengage, minimum throttle, high-speed disable, and lock release ramp.
Speed Mode

Speed mode applies requested lock based on vehicle speed alone.

  • Add, edit, or delete speed points.
  • Each point contains a speed bin and requested lock percentage.
  • The controller interpolates between points while driving.
  • Use Load, Defaults, and Save Curve to manage the curve stored in memory.
Throttle Mode

Throttle mode applies requested lock based on pedal or throttle value.

  • Use lower values for light throttle behavior and higher values for aggressive throttle input.
  • Each point contains a throttle bin and requested lock percentage.
  • This mode is useful when you want AWD response tied directly to driver demand.
  • Mode Settings still apply after the throttle curve is evaluated.
RPM Mode

RPM mode applies requested lock based on engine speed.

  • Each point contains an RPM bin and requested lock percentage.
  • This mode can shape AWD behavior around engine operating range instead of vehicle speed or pedal input.
  • Values are interpolated between saved RPM points.
  • Use Save Curve after edits so the controller keeps the new curve.
CAN View

CAN View is for finding, validating, and capturing signals. It is not intended to replace a dedicated high-speed CAN interface.

  • Filter decoded signals by ID or name, and choose chassis, Haldex, or all buses.
  • Decoded Frames show DBC-decoded values; Raw Frames show ID, DLC, data bytes, and age.
  • Use Download 30s dump when you need a short capture for debugging or sharing.
  • Diagnostic Capture temporarily switches the controller into a capture-friendly state.
  • Row colors indicate RX, TX, and generated frame edits.
Diagnostics

Diagnostics exposes live controller health, CAN state, Haldex telemetry, and generated frame information.

  • Use it to confirm chassis and Haldex CAN are present.
  • Check current mode, effective mode, Haldex generation, and frame diagnostics.
  • Use frame and telemetry values to verify whether control requests match what the vehicle is reporting.
Logs

Logs are for reviewing saved controller output and enabling debug streams when troubleshooting.

  • Use log files when reproducing a setup, CAN, or calibration issue.
  • Enable only the logging categories you need; extra logging can add overhead.
  • Download or clear logs after you have captured the information you need.
OTA

OTA handles firmware uploads and Wi-Fi configuration.

  • Upload a firmware .bin only when you are ready to update the controller.
  • STA settings connect the controller to a phone hotspot or local Wi-Fi network.
  • AP settings control the controller access point password.
  • Use Update Check to see whether the controller can reach update metadata.
